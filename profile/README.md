# Gerard Braad @ Red Hat

#### Jan 2017 - July 2026 · 9+ years

## Manager, Software Engineering

Led distributed engineering teams (11 engineers across China, India, and Europe) building CRC/OpenShift Local and JKube, serving thousands of developers globally.

### Products
  * [CRC](https://github.com/crc-org)/OpenShift Local  
    Red Hat development tool for provisioning a full OpenShift in ~10 minutes across Linux, macOS, Windows. Cross-platform virtualization without admin privileges.
  * Podman Infrastructure
    Networking and virtualization powering Podman Machine/Desktop. End-to-end Windows enablement.
  * [Eclipse JKube](https://www.eclipse.org/jkube/), [source code](https://github.com/eclipse/jkube), [@eclipse-jkube](https://github.com/eclipse-jkube)  
    Cloud-native Java tooling for Kubernetes.

#### Execution Excellence
100+ consecutive monthly releases over 9 years. Maintained delivery while re-architecting platform, adding new OS support, and coordinating with multiple teams.

#### Apple Silicon (2021)
Delivered full OpenShift on ARM Macs within 12 months of M1 announcement.

#### COVID Response (2020)
Rapidly optimized for remote work. Enabled thousands to work from home without business disruption.

#### Platform Evolution
Re-architected CRC to align with major OpenShift changes while maintaining a monthly release schedule.

#### Windows Ecosystem Integration
Led comprehensive Windows enablement across CRC, Podman, and Podman Desktop: installer integration, virtualization drivers, Desktop extensions, unified developer experience.

#### Cross-Platform Engineering
Built virtualization stack for Linux (KVM), macOS (Hyperkit/Apple framework),

#### Technologies
Kubernetes, OpenShift, Podman, Go, Python, Virtualization, Linux, macOS, Windows; (Hyper-V) without admin requirements. Solved complex OS-level constraints.
